# OpenStock Alerts（桌面端）使用手册

本应用用于“本地运行”的美股提醒与筛选：你设置好信息源（目前推荐 FMP；也兼容 Finnhub）与通知方式（Email/Webhook），再添加规则，应用就会按定时计划自动检查并触发提醒。

---

## 1. 你将得到什么

- **美股筛选（Screener）**：默认从全量美股候选池里筛选，也支持导入/粘贴 Symbols。
- **财报筛选（Financials）**：基于 FMP Premium 的最近季度财报数据，筛出营收增速、毛利率、EBITDA、现金流更好的公司。
- **提醒规则（Alerts）**：对一组 Symbols 设置价格/涨跌幅/均线/RSI/成交量放大等规则。
- **定时任务（Scheduler）**：支持按间隔执行，或每天固定时间执行（可选仅工作日）。
- **通知（Notifications）**：支持 Email（Gmail）或 Webhook（对接飞书/企业微信/Telegram 等）。
- **无需前端/无需登录**：所有数据与运行状态都保存在本地。

---

## 2. 快速开始（推荐流程）

1) **打开应用**  
解压你生成的 zip 包，双击运行 exe。

2) **先配置信息源与通知**  
进入「配置」页：
- 数据源：推荐先选 `FMP`
- FMP API Key：粘贴你的 Key
- FMP Base URL：默认即可（`https://financialmodelingprep.com`）
- 默认收件人：填你自己的邮箱（例如 `you@example.com`）
- Gmail 用户 + Gmail App Password：用于发邮件（建议使用 Gmail 的 App Password）
- 默认 Webhook（可选）：如果你想把提醒推到机器人/群聊，填 Webhook 地址

点击「保存配置」。

3) **设置定时任务**  
进入「定时」页：
- 模式：建议先用「每隔 N 秒」调试
- 间隔：例如 60（每分钟跑一次）
或使用「每日定时」：
- 时间：例如 `09:30`
- 仅工作日：是（推荐美股）

点击「保存定时配置」。

4) **先直接使用默认规则**  
进入「规则」页后，如果你还没有保存过规则，系统会自动给你一条默认规则：
- 最近一个成交日收盘市值 > 100 亿美元
- 最近一个成交日成交额 > 5 亿美元
- 5 个交易日内股价创历史新高

建议先不要改，直接用它验证 FMP 是否跑通。

5) **先 Dry-run 验证，再真实运行**  
进入「运行」：
- 点击「Dry-run 跑一次」：只计算，不发送通知，日志会显示命中情况
- 确认正常后点击「启动常驻」：应用将按定时计划持续运行

## 3. 财报筛选（Premium）怎么用

进入「财报」页后，先理解这页是做什么的：

- 它不是看当天股价波动
- 它看的是公司最近季度的财报质量
- 当前首版主要筛这些指标：
  - 最近季度营收同比
  - 最近季度毛利率
- 最近季度 EBITDA
- 最近季度 EBITDA 同比
- 最近季度 EBITDA 利润率
  - 最近季度经营利润率
  - 经营现金流是否为正
  - 自由现金流是否为正
  - 负债权益比是否过高

### 3.1 推荐用法

1. 先确保「配置」页已填好 `FMP API Key`
2. 进入「财报」页
3. 点击「套用成长财报模板」
4. 先把扫描数量设为 `50` 到 `100`
5. 点击「运行财报筛选」
6. 看结果表是否符合你的直觉
7. 如果结果满意，再点「将结果加入提醒池」

### 3.2 成长财报模板默认含义

- 市值至少 `10` 亿美元
- 最近季度营收同比至少 `15%`
- 最近季度毛利率至少 `40%`
- 最近季度 `EBITDA 同比`、`EBITDA 利润率` 都可额外设门槛；留空表示不限制
- 最近季度经营利润率至少 `10%`
- 经营现金流必须为正
- 自由现金流必须为正
- 负债权益比不高于 `2`

### 3.3 使用建议

- 第一次不要扫太多，先用 `50-100` 支验证
- 如果命中太少，就先下调毛利率或营收增速门槛
- 如果命中太多，就提高营收增速、经营利润率，或收紧负债权益比
- 财报数据更新不像股价那么频繁，所以财报筛选更适合做“找值得跟踪的公司”，而不是盘中提醒
- 全量美股模式下，候选池会按市值从高到低固定排序并保存到本地；后续默认复用这个顺序

### 3.4 AI 解读（DeepSeek v4）

- 财报结果表每一行现在都可以点 `AI解读`
- AI 只负责解释结构化财报结果，不参与筛选是否通过
- 配置入口在 `配置` 页：
  - `DeepSeek API Key`
  - `DeepSeek Base URL`
  - 模型：`deepseek-v4-flash` / `deepseek-v4-pro`
  - 思考模式：开 / 关
  - 思考强度：`high` / `max`
- 如果你更看重速度和成本，先用 `flash`
- 如果你更看重分析质量，再切到 `pro`

## 4. 规则怎么写（非程序员版）

你不需要写代码，只需要理解“规则 = Symbols + 条件 + AND/OR + 冷却 + 通知”。

### 3.0 标的范围：不用手填全市场

在「添加规则」弹窗里，“标的范围”支持两种方式：

- **手动填写**：适合只盯少量股票
- **全量美股（按市值从高到低，固定前N）**：不需要手工输入每个代号。你只需要设置筛选条件（例如 Min Price / Min Market Cap / Min Volume Ratio），系统会在全量美股里按条件筛出候选，再对候选执行提醒规则。

注意：全量美股数量很大，系统会按市值从高到低固定排序，并在每次运行时固定扫描前 N 支（默认 2000）。如果你想覆盖更多标的，就直接调大扫描上限（但会更慢/更容易触发限流）。

### 3.1 常见条件类型

- **价格 ≥ / ≤**：例如 `价格 ≥ 200`
- **涨跌幅 ≥ / ≤（%）**：例如 `涨跌幅 ≤ -3`（表示下跌 3% 以上触发）
- **上穿 / 下穿 SMA20**：价格与 20 日均线的“交叉”触发（不需要阈值）
- **RSI14 > / <**：常用超买超卖阈值（例如 `< 30`）
- **成交量放大 ≥（倍）**：当日成交量 ÷ 20 日均量（例如 `≥ 2` 表示放量至少 2 倍）
- **市值 ≥（百万美元）**：例如 `10000` 表示至少 100 亿美元
- **最近成交日成交额 ≥（百万美元）**：例如 `500` 表示至少 5 亿美元
- **5个交易日内股价创历史新高**：不用填阈值

### 3.2 AND / OR 的区别

- **AND（满足全部）**：所有条件都满足才触发  
例：`跌幅 ≤ -3% AND 成交量放大 ≥ 2`（常用于“放量下跌/放量拉升”）

- **OR（满足任一）**：任意一条满足就触发  
例：`价格 ≥ 200 OR RSI14 < 30`（更宽松）

### 3.3 冷却时间是什么

同一条规则命中后，在冷却时间内不会重复提醒。  
例如冷却 3600 秒：同一规则对同一 symbol 1 小时最多提醒一次。

## 5. 美股筛选（Screener）怎么用

进入「筛选」页，你有两种导入 Symbols 的方式：

### 4.0 默认全量美股（推荐）

筛选页默认选择「全量美股（按市值从高到低，固定前N）」作为标的来源：应用会自动维护一份“全量美股标的列表”，你不需要手工填写每个标的。

注意：全量美股数量较大，考虑到数据源限流与运行速度，筛选会固定扫描按市值排序后的前 N 支。你可以直接调大“本次扫描数量”来扩大覆盖面（但会更慢/更容易触发限流）。

### 4.1 导入 CSV

- 点击「导入 CSV」
- CSV 的**第一列**会被当作 symbol（例如你现有的 `screener_results.csv`）

### 4.2 直接粘贴

在「Symbol 列表」中每行一个：
```
AAPL
MSFT
NVDA
```

### 4.3 设置筛选条件

- Min Price：最低股价
- Min Market Cap：最低市值（单位：百万美元）
- Min Volume Ratio：最低成交量放大倍数（例如 2）

点击「运行筛选」后会出表格结果。  
点击「用结果创建规则」可以把筛选结果直接转成一条提醒规则（你再按需补充条件）。

## 6. 通知配置（Email / Webhook）

### 5.1 Email（Gmail）

你需要在「配置」里填写：
- Gmail 用户：`your@gmail.com`
- Gmail App Password：Gmail 生成的 16 位应用专用密码（推荐，不要用网页登录密码）
- 默认收件人：你的接收邮箱

注意：Email 的发送是“可选”的，如果你没填邮箱配置，规则仍然可以跑，只是不会发邮件。

### 5.2 Webhook

如果你有机器人/群聊 Webhook（例如飞书/企业微信/自建服务），填到：
- 配置页的「默认 Webhook」，或
- 在单条规则里填写「Webhook(可选)」覆盖默认值

Webhook 发送的是 JSON，方便你在下游做二次处理。

## 7. 运行与排障（常见问题）

### 6.1 运行按钮说明

- **Dry-run 跑一次**：不发送通知，用于验证配置与规则是否能正常计算
- **真实跑一次**：立即跑一轮，并按规则发送通知
- **启动常驻**：按定时计划持续运行（建议放后台运行）
- **停止**：停止常驻

### 6.2 常见问题排查

- **没有触发提醒**：先 dry-run，看日志是否能拉到行情数据；检查 Symbols 是否正确（如 AAPL）。
- **FMP 报错**：确认数据源已切到 `FMP`，API Key 正确，且套餐支持你调用的 endpoint。
- **Finnhub 报错**：确认 API Key 正确且没有超出限流；必要时把轮询间隔调大。
- **邮件发不出去**：确认使用 Gmail App Password；邮箱/密码填错会在日志里看到错误。

你可以备份这些文件来迁移配置。
